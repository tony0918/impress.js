You should know JIRA

You should know


Atlassian是一家2002年在澳大利亚成立的软件公司，该公司始终围绕一个主题，如何更好的为企业管理issue和知识管理，提高软件企业开发和管理的能力，制定相应的标准，方便管理者清楚的了解整个项目的过程和进度。除了JIRA之外，还有Confluence,HipChat,Bitbucket,Stash,Fisheye等众多的软件开发管理的相关工具。

Some Keywords
  1. Confluence   企业知识管理（WIKI）
  2. HipChat   企业内部及时沟通工具（OC like）
  3. Bitbucket   公有云代码托管（Github like）
  4. Stash   企业私有代码托管Server（GitLab like）
  5. Fisheye   企业私有代码review工具（Perforce like）
  6. Crowd   企业用户管理平台（SingleSignOn）
  7. Bamboo   企业发布持续集成工具（Maven like）
  8. SourceTree   免费Git图形化见面工具
JIRA可以做什么
  1. 通过Userstory来记录用户的需求
  2. 为每个Userstory开设不同的Task
  3. 通过Task来指定实施人员和跟进进度
  4. 通过GREENHOPPER插件来管理Scrum敏捷开发

Answer 1

需求直接转变成Userstory

Answer 2

通过电子化公示板展示项目Task所处的阶段

Answer 3

通过不同的过滤器来筛选Task列表

Answer 4

通过图表和系统归纳为项目评审阶段提供可参考的依据

JIRA的优势

有专业的团队为其更新

拥有充足的插件来管理项目整个阶段

代码管理，Code Review等都不是问题，需要按照Fisheye插件

etc. 多如牛毛

所以，只要承担得起，那么最适合的就是JIRA

最佳实践
  1. 在开需求会议时，客户提到的需求尽可能开设多的Userstory
  2. 在项目规划阶段，为每个Userstory详细考虑，结合自身团队的实际情况，开设尽可能详细的Task
  3. 在项目Kickoff阶段，合理分配每个sprint该完成的Task数量，并设立相应的milestone
  4. 每天按照权重来排序下还没有完成的Task，并设立Due Date
